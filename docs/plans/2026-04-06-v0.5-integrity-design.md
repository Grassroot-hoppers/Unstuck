# Unstuck v0.5.x — Integrity & consistency design

**Date:** 6 April 2026  
**Status:** Approved direction (product policy below); implementation plan follows separately.

## Product policy — facet reliability (replaces “rejection”)

**No facet is excluded from the engine.** Facets with **Cronbach’s α &lt; .70** (or the project’s chosen reliability cutoff) are **flagged in the UI and prose as lower-confidence measurement**, but their **raw scores, percentiles, domain aggregates, scatter, and rule inputs** are **always used** like any other facet.

Rationale:

- Matches how many IPIP-based tools treat weaker facets: warn, don’t silently drop.
- Removes the internal inconsistency where copy said “disregard” while maths still mixed different definitions.
- Aligns domain headlines and facet tables with a single definition: **full 30-facet IPIP-NEO-120 structure**.

**User-facing language (intent):**

- Replace “rejected” / “safer to disregard” with wording such as **below ~70% internal consistency (α)** — **interpret cautiously**, not “invalid” or “unused”.
- Stats summaries should count **low-reliability facets**, not “rejected” facets.

**Code terminology (recommended):**

- Deprecate `REJECTED_FACETS` and any `isRejected` flag that implies exclusion.
- Prefer `getConfidenceTier(facet)` (or equivalent) where the lowest tier (today `rejected`) is renamed to something like **`caution`** or **`low_reliability`** — still α &lt; .70, but not “out of the model.”

## Goals (v0.5.x scope)

1. **Normative alignment:** Facet means/SDs/α in `normative-data.js` match the **cited** Kajonius & Johnson (2019) source **or** the citation/attribution is updated to whatever source actually generated the numbers (no Table A1 claim unless verified).
2. **Single scoring story:** Domain score = **mean of all six facet raw scores** per domain; domain percentile from `DOMAIN_NORMS` for that same composite; **no** parallel unused “mean of percentiles” return path unless deliberately specified and tested.
3. **Reliability = highlight only:** Scoring and rules **never** return `null` percentiles solely because α is low; `DOMAIN_FACETS` lists **all six** facets per domain.
4. **Copy fixes:** `flag_N3_high` inverted low-pole sentence; `flag_N6_high` singular “1 person”; remove internal jargon (“core signal”, etc.) from user-visible strings where still present; optional tightening of “room of 100” complementarity.
5. **Scatter / extreme bands:** Revisit scatter thresholds once domains return to six facets (calibration against audit BUG-6 is simpler than under mixed facet counts).

## Non-goals (v0.5.x; see v0.6 handoff)

- Age×gender stratified norms.
- Large new rule catalog, teaser diversity overhaul, full research-block coverage.
- Symmetric percentile bands (optional follow-up).

## Architecture changes (by module)

| Area | Change |
|------|--------|
| `normative-data.js` | Reconcile numbers with paper; **remove** `REJECTED_FACETS`; expand `DOMAIN_FACETS` to six facets per domain; keep `getConfidenceTier` but rename lowest tier for clarity. |
| `scoring.js` | Always compute percentiles for every facet in `NORMS`; domain raw = mean of six facets; drop exclusion branches; remove or unify dead paths; return metadata for **low-reliability** facet list if useful for UI (optional). |
| `rules.js` | Remove “skip if facet rejected”; keep `ruleTier` for confidence of fired rules (weakest facet tier), using renamed lowest tier. |
| `engine.js` | Facet rows: always have percentile + `roomOf100`; flag low reliability via tier, not exclusion; rename `isRejected` → e.g. `isLowReliability` or derive from tier only. |
| `results.html` | Badge copy and CSS class names: “low reliability / cautious interpretation”, not “disregard”. |
| `interpretive/*.js` | Replace checks for tier `rejected` with `caution` (or chosen name); update tests (`tiers.test.js`, `build-report.test.js`). |
| `ipip-neo-120.config.js` | Export low-reliability facet list by α threshold instead of `REJECTED_FACETS`. |

## Data flow (target)

1. Raw → facet percentile for **all** facets with norms (clamp 1–99 as today).
2. Domain percentile ← z-score of **mean(six raw scores in domain)** vs `DOMAIN_NORMS`.
3. Scatter ← max − min of **six** facet percentiles in that domain.
4. Rules ← all facets addressable; confidence on a fired rule = weakest tier among its facets.
