# v0.5 integrity — implementation plan

> **For agentic workers:** Execute tasks in order. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Align norms with the cited literature; remove facet **exclusion** so α &lt; .70 facets are **used everywhere** but **labeled** as low reliability; fix scoring/domain/scatter consistency; patch known template/UI copy bugs.

**Architecture:** `normative-data.js` remains the single source for means, SDs, α, domain norms, and per-domain facet lists (six facets each). `getConfidenceTier` drives UI “reliability” styling only. `scoring.js` never nulls percentiles for reliability reasons. Rules evaluate on full percentiles.

**Tech stack:** Node test runner, existing `src/engine/*.js`, `src/results.html`, `src/test-personas.js`.

**Design reference:** [2026-04-06-v0.5-integrity-design.md](./2026-04-06-v0.5-integrity-design.md)

**Test (add if absent):**

```js
import test from 'node:test';
import assert from 'node:assert/strict';
import { scoreProfile } from './scoring.js';

test('all normed facets get numeric percentiles', () => {
  const raw = {};
  for (let i = 1; i <= 6; i++) {
    raw[`N${i}`] = 12;
    raw[`E${i}`] = 12;
    raw[`O${i}`] = 12;
    raw[`A${i}`] = 14;
    raw[`C${i}`] = 14;
  }
  const p = scoreProfile(raw);
  assert.ok(p.percentiles.N5 != null);
  assert.ok(p.percentiles.E4 != null);
  assert.ok(p.percentiles.O3 != null && p.percentiles.O6 != null);
  assert.ok(p.percentiles.C3 != null);
});
```
